1. WEB APPLICATION OVERVIEW
================================================================================

WHAT IS THIS PLATFORM?
----------------------
The Church Digital Engagement Platform is a production-ready digital content 
management and community engagement system designed specifically for churches 
and faith-based organizations. It serves as the central hub for delivering 
spiritual content, facilitating member interaction, and managing church 
communications.

PRIMARY PURPOSE:
- Deliver spiritual content (sermons, devotionals, articles, announcements)
- Enable community engagement through comments, reactions, and discussions
- Manage member registration, authentication, and roles
- Facilitate church communications via email campaigns and newsletters
- Provide content moderation and community management tools
- Track engagement analytics and content performance

TARGET USERS:
- Public visitors (non-authenticated)
- Church members (authenticated users)
- Administrators & Moderators (staff/leadership)

TECHNOLOGY STACK:
- Backend: Django REST Framework API
- Frontend: React with TypeScript
- Authentication: JWT-based (access + refresh tokens)
- Database: PostgreSQL (production)

CORE PHILOSOPHY:
- Production system (not a demo or prototype)
- Backend is the single source of truth
- Role-based access control enforced at API level
- Security-first approach
- Professional, church-appropriate design
- Calm, accessible user interface

2. PUBLIC PAGES (NON-AUTHENTICATED VISITORS)
================================================================================

WHO ARE PUBLIC USERS?
---------------------
- First-time visitors to the church website
- Community members exploring church content
- Non-members interested in sermons/articles
- Anyone with internet access (no login required)

WHAT PUBLIC USERS CAN ACCESS:
-----------------------------

HOMEPAGE / MAIN LANDING PAGE:
- Featured content showcase (highlighted sermon, article, or announcement)
- Browse recent content organized by type:
  * Sermons (recorded teachings/messages)
  * Articles (written teachings/reflections)
  * Announcements (church news and updates)
  * Devotionals (daily spiritual reflections)
  * Series (multi-part content collections)
- View content details (title, excerpt, author, publish date)
- See engagement metrics (view counts, reaction counts, comment counts)
- Filter/browse content by category
- Search for specific content
- Featured scripture quotes and reflections
- Call-to-action to register/login

CONTENT VIEWING:
- Read full articles and devotionals
- Listen to/watch sermons (if media is embedded)
- View content metadata (author, date, type, engagement stats)
- See content organized in series
- Browse related content

CURRENT HOMEPAGE IMPLEMENTATION STATUS:
- Multiple homepage files exist (HomePage.tsx, HomePage.css, HomePage.redesign.css)
- Current implementation is described as "messy" and not matching professional design standards
- Contains dynamic featured content with real database integration
- Displays various content sections (sermons, series, articles)
- Shows interactive elements (view counts, reactions)
- Includes navigation categories and content filtering
- Has scripture reflection quotes
- Current design uses emojis as icons (üéôÔ∏è for sermons, üìù for articles, etc.)

WHAT PUBLIC USERS CANNOT DO:
- Comment on content
- React to content (like, amen, love, praise reactions)
- Ask questions
- Subscribe to email campaigns
- Access member-only content
- View personal dashboards
- Edit or create content

PUBLIC USER JOURNEY:
1. Land on homepage and see featured/recent content
2. Browse content by type or search for topics
3. Read/view content of interest
4. Prompted to create account to engage (comment/react)
5. Register to become a member

- Personal profile management
- Email subscription preferences

4. ADMIN PAGES (ADMINISTRATORS & MODERATORS)
================================================================================

WHO ARE ADMINS/MODERATORS?
--------------------------
- Church staff and leadership
- Content creators and publishers
- Community moderators
- System administrators
- Users with elevated permissions

ROLE DISTINCTION:
- ADMIN: Full system access, can manage all content and users
- MODERATOR: Can create/manage content, moderate interactions, limited user management

WHAT ADMINS CAN ACCESS:
-----------------------

ADMIN DASHBOARD:
- System-wide statistics and analytics:
  * Total content pieces by type
  * User counts by role
  * Engagement metrics (views, comments, reactions)
  * Recent activity timeline
- Quick action shortcuts
- Content performance insights
- User growth metrics
- Email campaign statistics

CONTENT MANAGEMENT:
- Create new content (posts):
  * Choose content type (Sermon, Article, Announcement, Devotional)
  * Rich text editor for body content
  * Add title, excerpt, featured image
  * Set author attribution
  * Manage tags and categorization
  * Upload media attachments
- Edit existing content
- Delete content
- Publish/unpublish content
- Schedule automatic publishing (set future publish date)
- Manage content visibility (public vs members-only)
- View content analytics (views, engagement)
- Manage content in draft state

DRAFTS MANAGEMENT:
- Create and save draft content (unpublished)
- Edit drafts before publishing
- Preview drafts
- Schedule drafts for auto-publishing
- Convert drafts to published content
- Delete unused drafts

SERIES MANAGEMENT:
- Create content series (multi-part collections)
- Add/remove posts from series
- Set series order and sequencing
- Manage series metadata (title, description, image)
- Set series visibility:
  * Public (all visitors)
  * Members Only
  * Hidden (admins only)
- Track series analytics

EMAIL CAMPAIGN MANAGEMENT:
- Create bulk email campaigns
- Compose email content (subject, body)
- Select recipient groups:
  * All subscribed users
  * Specific content preferences (sermon recipients, etc.)
- Schedule email sends
- Track campaign status (draft, scheduled, sent)
- View campaign statistics (sent count, delivery status)
- Manage email templates

INTERACTION MODERATION:
- View all comments across all content
- Review flagged/reported comments
- Moderate inappropriate content:
  * Delete comments
  * Approve/reject pending comments
- Respond to user questions:
  * View questions marked by users
  * Provide official responses
  * Mark questions as answered
- Track question status:
  * Pending
  * Answered
  * Resolved
- View interaction analytics

USER MANAGEMENT (Admin only):
- View all registered users
- Search and filter users
- View user details:
  * Email, name, registration date
  * Email verification status
  * Account status (active/suspended)
  * Role assignment
- Manage user roles:
  * Change user role (Member, Moderator, Admin)
- Suspend/reactivate user accounts
- View user activity and engagement history
- Manage email subscription status

AUDIT LOGS (Admin only):
- Track all administrative actions
- View audit trail with:
  * Who performed the action
  * What action was performed (create, update, delete, publish, etc.)
  * When it occurred
  * What object was affected
  * Description of changes
- Action types tracked:
  * Content creation/editing/deletion
  * Publish/unpublish actions
  * User suspensions/reactivations
  * Role changes
  * Email campaigns sent
  * Comment deletions
  * Question answers
- Filter logs by action type, user, date range

APPLICATION SETTINGS:
- Configure system-wide settings
- Manage content types
- Configure email SMTP settings
- Set theme preferences (light/dark mode)
- Manage site metadata


ADMIN INTERFACE CURRENT STATE:
- Current design described as "unacceptable" and requiring complete rebuild
- Issues identified:
  * Emoji-based UI elements
  * Poor layout hierarchy
  * Non-responsive sidebar/topbar
  * Looks like a tutorial/beginner project
  * Not professional or presentation-ready
- Architecture includes:
  * Topbar with page navigation, theme toggle, user menu
  * Collapsible sidebar with navigation
  * Main content area for data tables and forms
- Target quality level: Enterprise SaaS (Stripe Admin, Vercel Dashboard, Contentful)

5. KEY FEATURES & FUNCTIONALITY ACROSS ALL USER TYPES
================================================================================

CONTENT TYPES SUPPORTED:
- Sermons: Recorded teachings/messages (audio/video)
- Articles: Written teachings and reflections
- Announcements: Church news, events, updates
- Devotionals: Daily spiritual reflections
- Series: Multi-part content collections

ENGAGEMENT FEATURES:
- View counters (track content popularity)
- Reaction system (like, amen, love, insight, praise)
- Commenting system (threaded replies supported)
- Question/answer system (moderated Q&A)

USER ROLES & PERMISSIONS:
- Visitor: Read-only access to public content
- Member: Read + Engage (comment, react, ask questions)
- Moderator: Member + Create/Edit content + Moderate interactions
- Admin: Full system access + User management + System settings

AUTHENTICATION SYSTEM:
- Email/password registration
- Email verification required
- JWT token-based authentication
- Secure password handling
- Password reset functionality

EMAIL SYSTEM:
- Campaign creation and scheduling
- Subscription management
- Preference-based delivery (choose content types)
- Bulk sending capability
- Delivery tracking

- Body (rich text)

- Content type (sermon, article, etc.)
- Featured image
- Tags/categories
- Publish date
- Status (draft, published, scheduled)
- Visibility (public, members-only)
- Engagement metrics (views, reactions, comments)

SERIES STRUCTURE:
- Series metadata (title, description, image)
- Ordered collection of posts
- Visibility settings
- Progress tracking

USER PROFILES:
- Email address
- First name, Last name
- Role (Member, Moderator, Admin)
- Registration date
- Email verification status
- Account status (active/suspended)
- Engagement statistics
